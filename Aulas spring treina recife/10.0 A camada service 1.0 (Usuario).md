A nossa camada service vai armazenar as regras de negócio da nossa API. Isso significa, que ao receber uma requisição do cliente, ela vai passar pelo controller (que será a próxima camada), e chegará ao service para passar pela lógica da nossa aplicação. Essa lógica vai decidir o que significará aquela requisição, e acionará um método do repository para mexer no BD.

Primeiro vamos criar nosso pacote de services. Depois, criaremos uma classe dentro dele para fazer as regras de negócio que serão aplicadas à nossa entidade Usuario.

![[{A33E83F8-5B33-48D1-B95A-9F3FD136BA84}.png]]

Primeiro vamos preparar nossa classe:

```Java
@Service  
public class UsuarioService {

	@Autowired
	private UsuarioRepository usuarioRepository;
	...
}
```

> `@Service` - Anotamos isso na nossa classe para que ela seja reconhecida pelo Spring como uma classe de serviço.

> `@Autowired` - Essa anotação serve para conectarmos as camadas da nossa API. Ao instanciar um objeto da `classe repository` (que criamos na camada anterior), podemos trazer os métodos daquela classe, que serão acionados usando os critérios dessa nossa classe Service.

Após isso, vamos começar definindo nossas operações báscias:

---
### Cadastrar um usuário / atualizar o cadastro

```Java
public Usuario salvarUsuario(Usuario usuario) {  
    return usuarioRepository.save(usuario);  
}
```

Declaramos o método nomeando-o `salvarUsuario`, nosso retorno será da classe `Usuario`, e passaremos um parâmetro também da classe `Usuario`, chamado `usuario`.

Retornaremos um método do objeto que injetamos com o `@Autowired`, o método será o `.save()`. Esse método não foi desenvolvido no nosso repository, ele foi herdado da classe `JpaRepository` que é mãe da nossa interface. É um método bem padrão.

Esse método recebe como parâmetro um objeto, que será o `usuario` que passamos como parâmetro no método `.salvarUsuario()` (esse método que estamos criando no service agora).

> **.save()** - Salvar dados novos no DB.

Isso vai enviar o objeto para o repostory, e por baixo dos panos o JpaRepository vai dar um `INSERT INTO TB_USUARIOS ... VALUES ...` no BD.

*Esse método também vai ser usado para atualizarmos Usuarios no BD. Na camada controller, vamos criar uma rota para esse método ser ativado quando for passado um usuário com ID que já existe, fazendo a sobreposição dos objetos.*

---
#### Listar todos os usuários cadastrados

Nesse método, precisamos que ele nos retorne uma lista de objetos da classe usuário.

```Java
public List<Usuario> listarUsuarios() {
    return usuarioRepository.findAll();  
}
```

Declaramos o método, tendo como retorno uma `List` de objetos da classe `Usuario`. O método não recebe parâmetro.

Para o retorno, usaremos um novo método da JpaRepository:

> `.findAll()` - Retornar todos os usuários do BD.

---
#### Buscar um usuário específico

```Java
public Usuario buscarUsuario(Long id) {  
    return usuarioRepository.findById(id).orElse(null);
}
```

Declaramos o método retornando um Usuario, e recebendo um Long, que será o ID do usuário a ser consultado que o cliente nos passará.

Usaremos o método:

> `.findById()` - Busca um usuario do dado via ID. Esse método não tem suporte para caso o ID passado no parâmetro não seja encontrado no BD, então precisamos complementá-lo para que ele retorne um `null` caso não encontre o id.

> `.orElse(null)` - Adiciona uma resposta caso a condição seja a de ID não encontrado.

---

#### Deletar um usuário

```Java
public void excluirUsuario(Long id) {  
    usuarioRepository.deleteById(id);  
}
```

> `.deleteById()` - Deleta o objeto correspondente àquele ID.
