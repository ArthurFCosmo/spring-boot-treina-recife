Criamos um pacote `dto` dentro da pasta do projeto para armazenar nossas classes de transferência de dados.

> `src\main\java\PASTA COM O NOME DO PROJETO\` criamos o pacote aqui.

---

## DadosUsuario

```Java
public record DadosUsuario(

    Long id,
    String nome,
    String email,
    String cpf,
    Integer idade,
    @JsonFormat(pattern = "dd/MM/yyyy")
    LocalDate dataNascimento,
    String Status

) {}
```

Repara nos detalhes:

- **`senha` não está aqui** — ela nunca vai aparecer na resposta da API.
- **`idade` está aqui** — mas não existe no banco de dados. Ela será **calculada** no Service antes de montar o DTO.
- **`@JsonFormat(pattern = "dd/MM/yyyy")`** — o banco guarda datas no formato americano (`2002-11-04`). Com essa anotação, o Jackson converte para o formato brasileiro (`04/11/2002`) ao montar o JSON de resposta.

## DadosProjeto

```Java
public record DadosProjeto(

    Long id,
    String nome,
    String descricao,
    @JsonFormat(pattern = "dd/MM/yyyy")
    LocalDate dataInicio,
    @JsonFormat(pattern = "dd/MM/yyyy")
    LocalDate dataConclusao,
    String status,
    String nomeResponsavel,
    List<DadosTarefa> tarefas

) {}
```

Aqui temos algo interessante: em vez de devolver o objeto `Usuario` inteiro como responsável, devolvemos apenas o `nomeResponsavel` — que é uma `String`. Assim o cliente recebe só o que precisa.

Também incluímos `List<DadosTarefa> tarefas`, o que significa que cada projeto na resposta já vem com a lista de tarefas vinculadas a ele.

## DadosTarefa

```Java
public record DadosTarefa(

    Long id,
    String titulo,
    String descricao,
    @JsonFormat(pattern = "dd/MM/yyyy")
    LocalDate dataCriacao,
    @JsonFormat(pattern = "dd/MM/yyyy")
    LocalDate dataConclusao,
    String prioridade,
    String status,
    Long idProjeto,
    String nomeProjeto,
    Long idUsuario,
    String nomeUsuario

) {}
```

Aqui também não devolvemos os objetos `Projeto` e `Usuario` completos — apenas os campos relevantes (`idProjeto`, `nomeProjeto`, `idUsuario`, `nomeUsuario`).

Note que `idUsuario` e `nomeUsuario` podem ser `null`, pois uma tarefa pode não estar atribuída a nenhum usuário.

## DadosLogin

```Java
public record DadosLogin(
    String email,
    String senha
) {}
```

Esse é o DTO de *entrada* (não de saída). Ele define o formato do corpo da requisição no endpoint de login — quem quiser se autenticar precisa enviar um JSON com `email` e `senha`.

---

> 💡 **Resumo:** cada DTO define exatamente o contrato de dados entre a API e o cliente — o que entra e o que sai, sem expor a estrutura interna do banco de dados.
