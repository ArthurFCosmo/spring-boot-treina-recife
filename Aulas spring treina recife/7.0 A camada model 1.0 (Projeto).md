Vamos explicar o que temos de novo em relação ao model de Usuario.

### Atributo: descricao

Declaramos esse atributo da seguinte forma:

```Java
@Column(columnDefinition = "TEXT")  
private String descricao;
```

> `@Column(columnDefinition = "TEXT")` - Nessa anotação, nós mudamos o formato do armazenamento de texto de VARCHAR para TEXT, que tem um limite maior do que 255 caracteres (cerca de 65.000 caracteres).

### Atributo: responsável

Esse atributo é diferente dos outros, pois ele recebe um objeto da classe Usuario, ao invés de datas ou Enums. Isso o torna especial pois ele fará menção a outra tabela, a de usuarios.

No BD, isso ele cria uma relação entre usuarios e projetos. Para isso, será necessária uma coluna de JOIN, que trará a chave estrangeira de usuario para projeto.

Essa coluna será justamente a coluna de "responsável", e a chave estrangeira será o ID do usuario.

Traduzindo isso para o Springboot, fica assim:

```Java
@ManyToOne  
@JoinColumn(nullable = false, name = "usuario_resp_id")  
private Usuario responsavel;
```

> `@ManyToOne` - Define a relação que essa coluna de JOIN vai ter no DB. No caso, manytoone significa que: um projeto pertence apenas a um usuário responsável / um único usuário pode ser responsável por vários projetos.
> 
> Sem essa anotação, o JPA não saberia que esse atributo Usuario deveria virar uma chave estrangeira que guardaria o ID do responsável.

> `@JoinColumn(nullable = false, name = "usuario_resp_id")` - Define a estrutura da nossa coluna que servirá de JOIN, assim como definimos em colunas comuns com @Column.
> 
> Primeiro definindo ela como `notnull`, depois dando um nome a ela, visto que o atributo é um objeto da classe Usuario, mas diferente das outras, o atributo não serve como nome da coluna, como em "e-mail", por exemplo. Então devemos definir o nome da coluna manualmente com  `name = "usuario_resp_id"`.
