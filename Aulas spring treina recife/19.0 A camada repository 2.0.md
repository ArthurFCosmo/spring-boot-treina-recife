Na v1.0, o repository era praticamente vazio — só herdava os métodos prontos do `JpaRepository`. Na v2.0, adicionamos **queries derivadas**: métodos com nomes especiais que o Spring interpreta e transforma em SQL automaticamente.

---

## Queries derivadas — o que são?

O Spring Data JPA tem um mecanismo de "derivação de query por nome de método". Você declara o método com um nome seguindo um padrão de palavras-chave, e o Spring gera o SQL correspondente sem que você precise escrever uma linha de SQL.

```Java
@Repository
public interface UsuarioRepository extends JpaRepository<Usuario, Long> {

    Optional<Usuario> findByCpf(String cpf);
    // → SELECT * FROM TB_USUARIOS WHERE cpf = ?

    Optional<Usuario> findByEmail(String email);
    // → SELECT * FROM TB_USUARIOS WHERE email = ?

    Optional<Usuario> findByEmailAndSenha(String email, String senha);
    // → SELECT * FROM TB_USUARIOS WHERE email = ? AND senha = ?

    List<Usuario> findByNomeContainingIgnoreCase(String nome);
    // → SELECT * FROM TB_USUARIOS WHERE LOWER(nome) LIKE '%?%'

    List<Usuario> findByDataNascimentoBetween(LocalDate dataInicio, LocalDate dataFinal);
    // → SELECT * FROM TB_USUARIOS WHERE data_nascimento BETWEEN ? AND ?

}
```

---

## Entendendo as palavras-chave

| Palavra-chave | O que faz no SQL                         |
| ------------- | ---------------------------------------- |
| `findBy`      | `WHERE`                                  |
| `And`         | `AND`                                    |
| `Containing`  | `LIKE '%?%'` (contém o valor)            |
| `IgnoreCase`  | `LOWER()` — ignora maiúsculas/minúsculas |
| `Between`     | `BETWEEN ? AND ?`                        |

Então `findByNomeContainingIgnoreCase(String nome)` vira:
```sql
SELECT * FROM TB_USUARIOS WHERE LOWER(nome) LIKE LOWER('%nome%')
```

---

## O tipo `Optional`

Vários métodos retornam `Optional<Usuario>` em vez de `Usuario`. `Optional` é um container que pode ou não conter um valor — é o Java dizendo "pode ser que exista, pode ser que não exista".

No service, usamos `.orElse(null)` para extrair o valor ou retornar `null` se não encontrar:

```Java
Usuario usuario = usuarioRepository.findByCpf(cpf).orElse(null);
```

Isso evita o `NullPointerException` que aconteceria se o Spring tentasse retornar um `Usuario` que não existe no banco.

## Queries derivadas no ProjetoRepository

```Java
@Repository
public interface ProjetoRepository extends JpaRepository<Projeto, Long> {

    List<Projeto> findByStatus(StatusProjeto status);
    // → SELECT * FROM TB_PROJETOS WHERE status = ?

    List<Projeto> findByResponsavel_Id(Long usuarioRespId);
    // → SELECT p FROM TB_PROJETOS p JOIN TB_USUARIOS u ON p.usuario_resp_id = u.id WHERE u.id = ?

}
```

Note o `findByResponsavel_Id`. O underscore (`_`) é usado para navegar entre entidades relacionadas:
- `Responsavel` é o atributo do tipo `Usuario` na classe `Projeto`
- `Id` é o atributo `id` dentro de `Usuario`

O Spring monta automaticamente o JOIN necessário entre as tabelas.

## Query derivada no TarefaRepository

```Java
@Repository
public interface TarefaRepository extends JpaRepository<Tarefa, Long> {

    List<Tarefa> findByProjeto_Id(Long projetoId);
    // → SELECT * FROM TB_TAREFAS WHERE projeto_id = ?

}
```

Mesma lógica do anterior — navega pelo atributo `Projeto` até o `id`.

---

> 💡 **Resumo:** queries derivadas permitem criar consultas SQL complexas apenas pelo nome do método, sem escrever SQL manualmente. O Spring lê o nome e gera a query automaticamente.
