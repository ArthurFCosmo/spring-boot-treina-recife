A camada repository funciona para definirmos os métodos de interação da nossa API com o BD.

Devemos criar um pacote na pasta do projeto para armazenar nossos repositories.

A base de tudo vem de fazer uma interface para ser o repository da nossa entidade, e fazer essa interface extender a classe JpaRepository (que já existe em um pacote padrão do spring), para puxar os métodos padrão dessa classe.

Devemos também marcar nossa classe com uma anotação `@Repository`, para o Springboot reconhecê-la como repository.

```Java
@Repository  
public interface UsuarioRepository extends JpaRepository<Usuario, Long> {  
}
```

Note que na extensão de `JpaRepository` passamos dois parâmetros.

Um deles é a Classe modelo que será alvo dos nossos métodos, e o tipo primitivo do Id dessa classe lá na camada model, que no nosso caso foi `Long`.

**Nessa parte 1.0 da nossa API, podemos replicar isso para projetos e tarefas. Futuramente voltaremos à camada repository para implementar nossos próprios métodos de utilização do BD.**
