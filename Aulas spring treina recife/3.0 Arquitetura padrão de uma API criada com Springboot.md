Ao desenvolver uma API utilizando o Springboot, utilizamos pacotes para dividir o backend em módulos.

- Model: É a parte que fica responsável por definir como vai ser o BD e as entidades que usaremos nele. Esse módulo cria elementos no banco de dados.

- Repository: No repository definiremos os métodos básicos que usaremos no BD: Select, Insert, Update, Delete. Esses métodos são herdados de uma classe específica, e serão chamados por outros módulos da API.

- Service: No módulo Service ficam as regras de negócio da API. É aqui que são feitos os cálculos e definidas as condições para se chamar um método do Repository para modificar os dados de fato no BD.

- Controller: É o módulo responsável por gerenciar as requisições e enviar as respostas ao cliente.
