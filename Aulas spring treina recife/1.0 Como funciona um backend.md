Na maioria dos projetos, o backend funciona seguindo esta arquitetura:

![[estrutura de uma aplicacao.png]]

Nesse esquema temos:

- Frontend: É a parte visual da aplicação, onde o Cliente opera. Ela funciona mandando requisições para o backend, por exemplo, sempre que o usuário clica em algum botão, uma mensagem é enviada para o backend processar o que deve acontecer em seguida, e depois retorna com o efeito sendo exibido no frontend.

- Backend: É a parte que "interliga o frontend com o banco de dados". O backend é responsável por uma série de funções, uma delas é receber as requisições do frontend, levá-las ao BD e devolvê-las. Ele também possui as regras da aplicação, que ditam os efeitos das ações do usuário. O backend também pode interligar a aplicação com API's externas, buscando e tratando dados para devolver ao front ou armazenar no BD.

- Banco de dados: É a parte da aplicação onde são armazenados os dados de maneira permanente.
