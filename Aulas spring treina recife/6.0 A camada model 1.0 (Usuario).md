### Anotações Spring

Agora vamos fazer o Srping trabalhar por nós. Com as anotações, podemos indicar pra ele como usar essa nossa classe no banco de dados.

#### Classe

Na nossa classe, anotaremos o seguinte:

```Java
@Data  
@NoArgsConstructor  
@AllArgsConstructor  
  
@Entity(name = "TB_USUARIOS")  
public class Usuario {

...

}
```

> `@Entity(name = "TB_USUARIOS")` - Anotação para fazer o Srping reconhecer essa classe como modelo de entidade e criar uma tabela dela no BD.

Também anotaremos na nossa classe:

> `@Data` - Anotação do lombok para criar getters e setters implícitos, executados em tempo de compilação
 
> `@NoArgsConstructor` - Anotação do lombok para criar construtor vazio (Exigido pelo Spring)

> `@AllArgsConstructor` - Anotação do lombok para criar construtor com os parâmetros.


- **Note que não é necessário ; no fim de anotações**

#### ID

No nosso atributo de ID, colocaremos duas anotações:

```Java
@Id 
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;
```

> `@Id` - Anotação para fazer o spring reconhecer o atributo a seguir como chave primária

> `@GeneratedValue(strategy = GenerationType.IDENTITY)` - Anotação para fazer o spring boot fazer o atributo a seguir ser auto gerado

#### Nome

Nos demais atributos, vamos anotá-los com `@Column()`, e dentro dos parênteses vamos colocar configurações para essas colunas na nossa tabela no BD.

No atributo "Nome", teremos o seguinte:

```Java
@Column(nullable = false, length = 150)
private String nome;
```

> `nullable = false` - É o notnull do SQL, faz com que o atributo (coluna) não possa ficar em branco, tem que receber um valor.

> `length = 150` - Faz com que o atributo tenha no máximo 150 caracteres, o padrão para varchar é 255.

#### CPF

```Java
@Column(nullable = false, length = 14, unique = true)
private String cpf;
```

> `unique = true` - Faz com que não o valor dessa coluna não possa se repetir para outra linha. (Não possam existir dois usuários com o CPF igual).

#### E-mail

```Java
@Column(nullable = false, unique = true) 
private String email;
```

#### Senha

```Java
@Column(nullable = false) // Anotação igual a de cima  
private String senha;
```

#### Data de nascimento

```Java
private LocalDate dataNascimento;
```

Sem anotações, por ser um atributo opcional.

#### Status

```Java
@Column(nullable = false) // Anotação igual a de cima  
@Enumerated(EnumType.STRING) // Anotação para dizer ao Spring que o atributo a seguir implementa um ENUM (Que stá em outro pacote)  
private StatusUsuario status;
```

> `@Enumerated(EnumType.STRING)` - Anotação para o Spring entender que o atributo implementa um ENUM (Que está em outro pacote). O `EnumType.STRING` define o tipo de conteúdo do ENUM.
