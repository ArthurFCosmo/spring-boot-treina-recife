## Criando um usuário (POST)

### URL
```
POST localhost:8080/usuarios
```

### Body (JSON)
```JSON
{
    "nome": "Arthur Cosmo",
    "cpf": "123.456.789-00",
    "email": "arthurcosmo@gmail.com",
    "senha": "1234",
    "dataNascimento": "2002-11-04",
    "status": "ATIVO"
}
```

### Resposta esperada (201 CREATED)
```JSON
{
    "id": 1,
    "nome": "Arthur Cosmo",
    "email": "arthurcosmo@gmail.com",
    "cpf": "123.***.***-**",
    "idade": 23,
    "dataNascimento": "04/11/2002",
    "status": "ATIVO"
}
```

Repara nas diferenças entre o que você mandou e o que voltou:
- O CPF voltou **mascarado**
- A data voltou no **formato brasileiro** (dd/MM/yyyy)
- A **idade** apareceu calculada automaticamente
- A **senha** não aparece na resposta

## Login (POST)

### URL
```
POST localhost:8080/usuarios/login
```

### Body (JSON)
```JSON
{
    "email": "arthurcosmo@gmail.com",
    "senha": "1234"
}
```

Se as credenciais baterem com algum registro no banco, retorna `200 OK` com o `DadosUsuario`. Se não encontrar, retorna `404 Not Found`.

## Buscando por nome (GET com @RequestParam)

### URL
```
GET localhost:8080/usuarios/busca?nome=Arthur
```

No Insomnia, você pode preencher os query params na aba **Params** em vez de digitar na URL manualmente. Cada linha é um par chave/valor.

Retorna uma lista de todos os usuários cujo nome contenha "Arthur" (sem diferenciar maiúsculas/minúsculas).

---

## Filtrando por data de nascimento (GET com @RequestParam)

### URL
```
GET localhost:8080/usuarios/filtro?dataInicio=2000-01-01&dataFinal=2005-12-31
```

Retorna todos os usuários nascidos entre 01/01/2000 e 31/12/2005.

> **Atenção:** o formato da data no `@RequestParam` deve ser `YYYY-MM-DD` (formato ISO, padrão do Java). O `@JsonFormat` que formata para `dd/MM/yyyy` só vale na **resposta** — não na URL.

## Criando um projeto (POST)

Para criar um projeto, você precisa passar o **objeto completo** do responsável no body — incluindo o ID do usuário já cadastrado:

### Body (JSON)
```JSON
{
    "nome": "SGP API",
    "descricao": "Sistema de Gestão de Projetos",
    "dataInicio": "2026-06-01",
    "dataConclusao": "2026-12-31",
    "status": "EM_ANDAMENTO",
    "responsavel": {
        "id": 1
    }
}
```

> O Spring Data JPA consegue resolver o relacionamento com apenas o `id` do responsável. Você não precisa mandar todos os dados do usuário.

## Criando uma tarefa (POST)

```JSON
{
    "titulo": "Implementar endpoint de login",
    "descricao": "Criar o POST /usuarios/login",
    "dataCriacao": "2026-06-01",
    "prioridade": "ALTA",
    "status": "EM_ANDAMENTO",
    "projeto": {
        "id": 1
    },
    "usuario": {
        "id": 1
    }
}
```

> O campo `usuario` pode ser omitido (tarefa sem responsável atribuído).

## Buscando projetos por status (GET)

### URL
```
GET localhost:8080/projetos/status/EM_ANDAMENTO
```

O valor do status vai direto na URL como `@PathVariable`. Use exatamente o mesmo nome que está no Enum.

---

## Buscando tarefas de um projeto (GET)

### URL
```
GET localhost:8080/tarefas/projeto/1
```

Retorna todas as tarefas vinculadas ao projeto de ID 1.

## Visão geral de todos os endpoints

### Usuários
| Método | URL | O que faz |
|---|---|---|
| `POST` | `/usuarios` | Cadastrar |
| `POST` | `/usuarios/login` | Login |
| `GET` | `/usuarios` | Listar todos |
| `GET` | `/usuarios/{id}` | Buscar por ID |
| `GET` | `/usuarios/cpf/{cpf}` | Buscar por CPF |
| `GET` | `/usuarios/email/{email}` | Buscar por e-mail |
| `GET` | `/usuarios/busca?nome=...` | Buscar por nome |
| `GET` | `/usuarios/filtro?dataInicio=...&dataFinal=...` | Filtrar por nascimento |
| `PUT` | `/usuarios/{id}` | Atualizar |
| `DELETE` | `/usuarios/{id}` | Deletar |

### Projetos
