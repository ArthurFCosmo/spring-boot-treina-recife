Na model de tarefas, teremos isso:

![[{AC9C1B79-B85C-4625-9816-BCB1C757556C}.png]]

Não temos muita coisa de diferente.

Temos dois atributos ENUM, que são status e prioridade.

### Atributo: usuario

```Java
@ManyToOne  
private Usuario usuario;
```

Temos um atributo FK (Chave estrangeira) que é o Usuario. Por ter definido ele apenas com `@ManyToOne`, ele poderá receber nulo, ou seja, uma tarefa pode não estar atrelada a nenhum usuário.

Também não definimos nome para ele com `@JoinColumn(name = "nome")`, então ele receberá o nome padrão, que é o **nome do atributo + id**. Resultando em `usuario_id`.

### Atributo: projeto

```Java
@ManyToOne  
@JoinColumn(nullable = false)  
private Projeto projeto;
```

Basicamente a mesma coisa da de cima, mas definimos ele como `notnull`, uma tarefa deve pertencer obrigatoriamente a algum projeto.

O nome dele também será automático, `projeto_id`.
