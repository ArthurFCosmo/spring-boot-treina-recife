Como ainda não temos um FRONTEND para funcionar como cliente nos enviando requisições, vamos utilizar uma ferramenta muito interessante chamada **Insominia**.

Ela funciona como um cliente emulado, o qual podemos criar requisições para testar nossa API.

No app, criamos uma nova coleção para organizar nossas requests.

Após isso, podemos iniciar com a criação.

## 🔷 Requisição para cadastrar um usuário

### URL

No campo do tipo da requisição, marcamos `POST`.

> `localhost:8080/usuario`

Utilizaremos a porta padrão da maioria dos BD's, visto que não definimos uma porta personalizada para ser utilizada. Na URL, utilizamos o endereçamento `/usuario`, como definimos na nossa classe `controller`.

### CORPO

No corpo, selecionamos a opção `JSON` e digitamos nosso `Usuario` baseado na nossa classe modelo lá na camada model.

> **Não colocaremos Id, pois ele é autogerado pelo BD.**

```JSON
{
	
	"nome": "Arthur Cosmo",
	"cpf": "123.456.789-00",
	"email": "arthurcosmo@gmail.com",
	"senha": "1234",
	"dataNascimento": "2002-11-04",
	"status": "ATIVO"
	
}
```

Após isso, basta enviar a request, e o Insominia retornará o status `201 CREATED`, como configuramos no controller, juntamente com o objeto que cadastramos.

## 🔷 Requisição para buscar todos usuários

### URL

No campo do tipo da requisição, marcamos `GET`.

> `localhost:8080/usuario`

Essa requisição não precisa de corpo, configuramos isso no controller.

Nos retornará um status `200 OK`, junto com uma lista de objetos `JSON`:

![[{38784720-DA43-49AA-92EE-8AD3B7904738}.png]]

### URL

> `localhost:8080/usuario/ID DO USUARIO`

## 🔷 Requisição para deletar um usuário

### URL 

No campo do tipo da requisição, marcamos `DELETE`.

> `localhost:8080/usuario/ID DO USUARIO`

Igualzinho ao de buscar por Id.

Se ele conseguir deletar um usuário existente ele retorna status `204 NO CONTENT`:

![[{912DB3DB-8616-4AC0-BB3E-470F9F656615}.png]]

Se ele não conseguir, retorna status `404 NOT FOUND` assim como no buscar por Id.

## 🔷 Requisição para atualizar um usuário

### URL 

No campo do tipo da requisição, marcamos `PUT`.

> `localhost:8080/usuario/ID DO USUARIO`

### Body

No body, colocaremos um `Usuario`, com todos os campos menos o `Id` assim como no `POST`.

```JSON
{
	
	"nome": "Glauber Batista",
	"cpf": "100.456.321-00",
	"email": "glauberbo@gmail.com",
	"senha": "1234",
	"dataNascimento": "2002-07-20",
	"status": "ATIVO"
	
}
```

> Se der tudo certo, ele retornará um status `200 OK`.

> Se nós colocarmos um Id que não existe na URL, retornará um status `404 NOT FOUND`.

> Se nós errarmos o Body que enviamos, podemos receber status `500 INTERNAL SERVER ERROR`. 
> 
> Por exemplo, se colocarmos os dados do **Glauber Batista** para serem atualizados, mas passarmos o **Id 1 (do Arthur Cosmo)**, teremos exatamente essa resposta, pois estará tentando cadastrar um CPF que já existe, mas marcamos o CPF como `unique` no nosso `model`.
